Configuration Email SMTP

Guide complet pour configurer l'envoi d'emails avec Gmail

Important : Gmail requiert un "Mot de passe d'application" pour l'authentification SMTP, pas votre mot de passe habituel.
1
Activer l'authentification à 2 facteurs

Rendez-vous sur votre compte Google et activez l'authentification à 2 facteurs si ce n'est pas déjà fait.

2
Générer un mot de passe d'application
  1. Allez dans Sécurité > Mots de passe des applications
  2. Sélectionnez Autre (nom personnalisé)
  3. Tapez Gestion de Budget
  4. Cliquez sur Générer
  5. Copiez le mot de passe à 16 caractères généré
3
Utiliser le mot de passe d'application

Ce mot de passe généré (format : abcd efgh ijkl mnop) remplace votre mot de passe Gmail habituel pour l'application.

Attention : N'utilisez jamais votre mot de passe Gmail principal dans le code. Utilisez uniquement le mot de passe d'application généré.
Configuration du fichier .env

Ajoutez ces variables à votre fichier .env :

# Email Configuration (SMTP) SMTP_HOST=smtp.gmail.com SMTP_PORT=587 SMTP_USER=votre-email@gmail.com SMTP_PASS=votre-mot-de-passe-application # Pour Gmail, utilisez un "Mot de passe d'application" généré
Exemple complet :
# Email Configuration (SMTP) SMTP_HOST=smtp.gmail.com SMTP_PORT=587 SMTP_USER=john.doe@gmail.com SMTP_PASS=abcd efgh ijkl mnop
Autres fournisseurs d'email :
Outlook/Hotmail
SMTP_HOST=smtp-mail.outlook.com SMTP_PORT=587 SMTP_USER=votre-email@outlook.com SMTP_PASS=votre-mot-de-passe
Yahoo
SMTP_HOST=smtp.mail.yahoo.com SMTP_PORT=587 SMTP_USER=votre-email@yahoo.com SMTP_PASS=votre-mot-de-passe-app
Tester la configuration
1
Test de connexion SMTP

Utilisez le script de test pour vérifier votre configuration :

cd "c:\\Users\\Youcode\\Desktop\\Gestion-de-Budget"
node scripts/testEmailConfig.js
2
Test via l'interface web

Testez directement sur la page de réinitialisation :

Tester la réinitialisation
3
Vérification des logs

Surveillez la console du serveur pour les messages :

  • ✅ SMTP connection verified successfully
  • ✅ Email sent successfully
  • ❌ SMTP connection failed
Dépannage :
  • Vérifiez que l'authentification 2FA est activée
  • Utilisez le bon mot de passe d'application
  • Vérifiez que le port 587 n'est pas bloqué
  • Essayez avec "Autoriser les applications moins sécurisées" (non recommandé)
